# 策衡项目协作约定

## 语言

- 默认使用中文沟通。
- 项目文档、设计说明、计划和注释默认使用中文编写。
- 代码标识符、包名、命令名和公开 API 使用英文。

## 项目信息

- 项目中文名为“策衡”。
- Go module 为 `github.com/ceheng-io/stock-go`。
- 根包名建议为 `stock`。
- 技术标识和 CLI 命令名为 `ceheng`。

## Go 代码约定

- 当前阶段先建设 Go 版本 SDK 库，Web 界面和后端服务后续再加入。
- 公共库 API 保持简洁稳定；数据源适配、请求治理和服务编排优先放在 `internal/` 下。
- 根包只做组合与薄委托，不直接写 provider 解析逻辑。
- 公开类型放在 `types/`，纯计算能力可放在 `indicators/`、`symbols/`、`signals/`、`screener/`、`cache/` 等公开子包。
- `types/` 按领域拆分公开类型文件，单个非测试 Go 文件不超过 1000 行；新增类型优先放入对应领域文件，超过限制继续拆分。
- 新增或修改 Go 代码后，优先执行 `go test ./...` 验证。

## 前端约定

- 写任何前端代码前，必须先调用或执行本地 `skills/frontend-design`。
- 根据 `skills/frontend-design` 的输出确认页面结构、交互状态、数据依赖和验收方式。
- 如果前端设计输入不足，先补齐设计问题，不要直接实现界面。
